================================================================================
                               DATA FLOW (STREAM)
================================================================================

1) User clicks PLAY
2) React → GET /stream
3) Express API receives request
4) Query PostgreSQL: SELECT * FROM tracks
5) Random track selection (Business Logic)
6) INSERT INTO history
7) fs.createReadStream(mp3)
8) Audio stream piped to HTTP response
9) Browser plays real-time audio 🎧


================================================================================
                           ARCHITECTURAL PATTERN
================================================================================

Frontend:        SPA (Single Page Application)
Backend:         RESTful API
Auth:            Token-based (JWT + Redis session store)
Database:        Relational Model (PostgreSQL)
Streaming:       HTTP Streaming (audio/mpeg)
Infrastructure:  Containerized (Docker)
